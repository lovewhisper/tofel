（句末的括号里面是一些随机的，针对文本的评论。目的是增加一点趣味性，也帮助大家对文章内容留下更深刻的印象。）



1) (female professor) Archeologists sometimes discover an unknown ancient written script(文本) and they need to decipher or figure out the meaning of the script in order to read and understand it.

考古学家有时会发现一种未知的古代文字，他们需要破译或弄清文字的含义，以便阅读和理解它。


2) And now, a script’s not the same as a language, right?

文字和语言是不一样的，对吗？


3) A script is a written symbolic communication system that’s associated with a spoken language.

文字是一种与口头语言对应的书面的符号交流系统。


4) So, the written symbols of a script can be used to represent the words, ideas of a language or languages.

所以，文字的书面符号可以用来代表一种或多种语言的词汇、思想。


5) [Now,] deciphering an ancient script is interesting to anthropologists because the script can both help us to reconstruct(重构) the language of a civilization and give us insight into that civilization’s customs(习俗) and daily life.

破译古代文字对人类学家来说很有意义，因为文字既能帮助我们重建一个文明的语言，又能让我们了解该文明的习俗和日常生活。


6) But traditionally, deciphering a script takes a long time.

但是用传统的方法，破译一种文字需要很长的时间。


7) However, a new computer program may help speed the process.

然而，一种新的计算机程序可能有助于加快这一过程。


8) To understand just how much let’s first look at traditional approach to deciphering one script, (1) a script that was deciphered(破译) only recently in the twentieth century, Ugaritic. 

为了理解破解过程能被加快多少，我们先来看看破译一种文字的传统方法，一种最近在二十世纪才被破译的文字，即UG。


9) Tablets with Ugaritic writing on them were found in Syria in the late 1920s.

20世纪20年代末，在叙利亚发现了带有UG文字的石板。


10) Now, when faced with an unknown script, you’d ask questions like, first, what type of script is this? 

现在，当面对一个未知的文字时，你会问这样的问题：首先，这是什么类型的文字？


11) There are two main types: either, well, each symbol represents a whole idea or word, or, each symbol(符号) represents a sound or a combination of sounds, the way an alphabet(字母表) does.

文字有两种主要类型：要么，每个符号代表一个完整的概念或单词，要么，每个符号代表一个声音或声音的组合，就像字母表那样。


12) In the case of Ugaritic, only thirty symbols were used on all the tablets, suggesting the symbols were alphabetic. (2-D)

在UG的例子中，所有的石板上只使用了30个符号，这表明这些符号是字母型的。（也就是表声的）


13) You’d also ask whether the language itself is related to any other known language.

我们还会问另外一个问题，这种语言是否与任何其他已知的语言有关。


14) And because of where these tablets were found and their age, we thought Ugaritic might be related to ancient Hebrew. (2-A)

而由于这些石板的发现地点和年代，我们认为UG可能与H有关。


15) This is important because we can use the known script as a clue to the unknown script. 

这一点很重要，因为我们可以把已知的文字作为未知文字的一个线索。


16) So, if we know a script’s alphabetic, we try to map its symbols onto the alphabet of the known script.

因此，如果我们知道一种文字是字母型的，我们就尝试将其符号与已知文本的字母联系起来。


17) But, this is no simple task. You have to figure out pieces one by one.

但是，这可是不容易的事，我们只能一点一点地弄清楚。


18) You may start out by mapping the unknown alphabet onto the known alphabet only to later learn it was an incorrect mapping. 

你可能一开始就把未知的字母映射到已知的字母上，后来才知道这是一个不正确的映射关系。


19) Or, you may just get stuck. And the study of Ugaritic was heading in that direction, until the discovery of five axes with writing on them. (3)

或者，我们的研究可能会止步不前。对UG语的研究正朝着这个方向发展（遇到了瓶颈），直到发现了有五把写有文字的斧头。


20) Each one had the same combination of six symbols and the fifth axe also had an additional four symbols.

每把斧子上的文字都由相同的六个符号组合，第五把斧头还有额外的四个符号。


21) Researchers guessed the written symbols common to all the axes would denote a name.

研究人员猜测，所有斧头上共有的文字符号将表示一个名字。


22) But the fifth axe, they believed that the four symbols that preceded the name would spell out the word axe in Ugaritic.

但对于第五把斧头，他们认为名字前面的四个符号会拼出UG语中的斧头一词。


23) What followed took a lot of work. But it turns out the ancient Hebrew word for axe has the same number of symbols and is quite similar.

接下来的工作花了很多时间。但事实证明，古希伯来语中的”斧头“这个词有相同数量的符号，而且相当相似。


24) These four symbols led to further discoveries and now we are mostly able to read Ugaritic.

这四个符号导致了进一步的发现，现在我们基本能够阅读UG语。


25) So, this traditional approach can work but it’s time-consuming.

所以，这种传统的方法是能用，但是很耗时。


26) [Now, recently,] computer scientists devised(设计) a program meant to decipher unknown scripts.

最近，计算机科学家设计了一个用来破译未知文字的程序。


27) They tested the computer program with Ugaritic to see if the program’s decipherment matched the human one.

他们用UG语测试了这个计算机程序，看看程序的破译结果是否与人类破译的结果一致。


28) Amazingly, the program deciphered Ugaritic in just a few hours. Compare that to years it took humans.

令人惊讶的是，程序只用了几个小时就破译了UG。相比之下，人类需要花费数年时间。


29) OK, the program itself. It works by quickly comparing the new script to a known script.

我们回到这个程序本身。它的工作原理是将新的文字与已知的文字进行快速比较。


30) So, what exactly does it compare?

那么，它到底是怎么比较的呢？


31) Well, it’s programmed to make certain assumptions about the features of the two scripts and languages. 

嗯，程序被设定对两种文字和语言的特征做了某些假设。


32) First, it assumes related letters or symbols in the scripts will appear about the same number of times with a similar frequency. (4-D)

首先，它假设两种文本中相关的字母或符号会以类似的频率出现。


33) And the second assumption concerns word structure, um, the way words are built. 

第二个假设是关于单词的构建方式。


34) Corresponding words in the related language might also have a similar structure. (4-A)

相关语言中的对应词可能也有类似的结构。


35) So, the system uses these assumptions together to attempt to decipher the script.

系统使用这些假设来试图破译文字。


36) (male student) [It’s kind of limited though, right?]

但是这种方法有局限性的，对吧？


37) Cause the program can’t decipher script unless there’s a known script to compare it to.

因为必须有一个已知的文本来做对比，不然程序就不能进行破译了。


38) (professor) It is. But remember, that’s a limitation no matter the approach.

是的。但是别忘了，不管是什么方法都有这种局限性。


39) If there’s no related language to compare it to, a script simply can’t be deciphered. (5) 

如果没有相关的语言进行比较，我们用什么方法都无法进行破译。


40) Take Etruscan, Etruscan dates to around 700 BCE in what’s present-day Italy.

以E语为例，E语可以追溯到公元前700年左右，一个位于现今意大利境内的地方。（时间地点通常都是过小的细节，不重要。）


41) It isn’t related to any known language as far as we know. So we can see the traditional approach won’t be successful.

就我们所知，它与任何已知的语言都没有联系。所以我们传统的方法不能破解它。


42) On the other hand, the computer program, well, it should soon be able to scan multiple scripts at a time to look for similarities.

另一方面，计算机程序应该很快就能一次扫描多个文本，以寻找它们相似之处。


43) And so it’s entirely possible that the program itself might someday discover a previously unknown relationship between two languages. (6)

因此，程序本身完全有可能在某一天发现两种语言之间以前未知的联系。
